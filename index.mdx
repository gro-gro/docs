## 1 · Introducción y Conceptos Básicos

### ¿Qué es Linear?

Linear es la herramienta donde centralizamos **todas** las tareas de Zeratype. Sirve para planificar, asignar, seguir y cerrar trabajo.

### ¿Qué es un Issue?

* **Definición:** Cualquier tarea accionable que consume tiempo o recursos.
* **Ejemplos:** Editar un video, diseñar una portada, coordinar una reunión.
* **NO son issues:** Ideas sueltas → van al **Backlog**.

### Roles en Zeratype

| Rol | Descripción | Responsabilidades clave |
| -------------- | -------------------------------------------------- | -------------------------------------------------------------------------------- |
| **Ejecutores** | Editores, diseñadores, técnicos, productores, etc. | Hacer el trabajo, auto‑asignarse issues pendientes, actualizar horas y status. |
| **Leads** | Responsables de proyectos. También ejecutan. | Crear y asignar issues, proveer materiales, revisar avances, destrabar bloqueos. |

## 2 · Flujo de Trabajo

### 2.1 Ciclo de vida de un Issue

```mermaid
flowchart LR
    A[To‑Do] --> B[En Edición]
    B --> C[Para Revisar]
    C -- Cambios? -->|Sí| B
    C -- OK --> D[Listo/Subido]
```

### 2.2 Status principales

* **To‑Do:** Listo para empezar (Lead lo crea).
* **En Edición:** En progreso (Ejecutor).
* **Para Revisar / Para Corregir:** Terminado y esperando feedback (Ejecutor) o necesita ajustes (Lead).
* **Listo/Subido:** Aprobado por cliente o Lead.

### 2.3 Nomenclatura de Issues

```
[InicialesProyecto]_[TipoContenido]_[Nombre]
Ej.: CL_CLIP_Maternidad
```

### 2.4 Relaciones entre Issues

* **Bloquea a:** Este issue impide que otros empiecen.
* **Bloqueado por:** No se puede iniciar hasta que el bloqueador termine.
  (Clic derecho en el issue → *Marcar como* → elegir relación.)

## 3 · Manual para Ejecutores

### 3.1 Antes de empezar

1. **Revisá la cola de issues `To‑Do`/`Bloqueados` de tu equipo cada mañana.**
2. Auto‑asignate lo que vas a tomar (atajo **I**). Si falta un issue, ¡créalo!

### 3.2 Crear un Issue rápido

1. **C** → *Template Rápida* (atajo **Alt + C**).
2. Completá:
   * Nombre siguiendo la nomenclatura.
   * Equipo.
   * Labels (CLIP, REEL, DISEÑO…).
3. Guardá (**Ctrl + Enter**).

### 3.3 Gestionar Status

* Al empezar → **S** → *En Edición*.
* Al terminar → **S** → *Para Revisar*.

### 3.4 Trackeo de horas

En la descripción agregar:

```
Horas: 8+8+3   ← bloques trabajados, sin espacios
```

> *Todos los issues deben tener horas cargadas.*

### 3.5 Shortcuts clave de Ejecutores

* **Crear issue:** `C`
* **Guardar:** `Ctrl + Enter`
* **Cambiar status:** `S`
* **Asignarme:** `I`
* **Establecer fecha límite:** `Shift + D`
* **Abrir menú comandos:** `Ctrl + K`
  (Ver sección [Shortcuts](#shortcuts) para la lista completa).

### 3.6 Buenas prácticas

* Todo lo que hagas debe tener su issue y horas.
* Comentá dentro del issue cualquier duda o material adicional.
* Usá la vista *Preview* (`Space`) para una revisión rápida.

## 4 · Manual para Leads

### 4.1 Creación y asignación de Issues

1. **Alt + C** → elegí la template correcta (EDI, DIS, PROD, TCN).
2. Completá **Equipo, Due Date, Estimate** y proveé materiales o referencias.
3. Asigná inicialmente o dejá en *To‑Do* para auto‑asignación del equipo.

### 4.2 Campos descriptivos

| Campo | Uso |
| ------------ | -------------------------------------- |
| **Proyecto** | Cliente o producción general |
| **Label** | CLIP, REEL, DISEÑO, etc. |
| **Estimate** | XS / S / M / L / XL (ver guía interna) |

### 4.3 Gestión de Status

* **To‑Do:** Issue listo (con todo el material).
* **Para Corregir:** Devolver feedback claro en la descripción.
* **Listo/Subido:** Cuando el cliente aprueba.

### 4.4 Trackeo de horas

Incluí siempre el campo **Horas:** en el template para que ejecutores completen.

### 4.5 Manejo de bloqueos

Usá relaciones **Bloquea a / Bloqueado por** para mostrar dependencias y evitar cuellos de botella.

### 4.6 Shortcuts clave de Leads

(Idénticos a los de Ejecutores; ver [Shortcuts](#shortcuts)).

### 4.7 Buenas prácticas

* Plantillas consistentes → menos errores.
* Checklist rápido antes de mover a *To‑Do*: ¿Material? ¿Due Date? ¿Estimate? ¿Bloqueos? ✔️
* Revisá progresos a diario y destrabá al equipo.

<a name="shortcuts"></a>

## 5 · Referencia de Shortcuts

### Issues

`C` Crear · `Alt + C` Desde template · `Ctrl + Enter` Guardar/Enviar · `S` Status · `Shift + D` Due Date · `L` Labels

### Navegación

`Ctrl + K` Comandos · `Ctrl + 1-9` Ir a equipo · `O T` Abrir equipo · `O I` Abrir issue · `O P` Abrir proyecto · `G P` Ir a proyecto

### Asignaciones

`I` Asignarme · `A` Asignar usuario

### Visualización

`Space` Preview · `Shift + Click` Selección múltiple

<a name="glosario"></a>

## 6 · Glosario & FAQ

* **Issue:** Tarea accionable.
* **Backlog:** Lista de ideas o tareas futuras.
* **Estimate:** Tamaño relativo de esfuerzo (XS‑XL).
* **Bloquea a / Bloqueado por:** Relación de dependencia.
* **Horas:** Campo en la descripción para sumar tiempo invertido.
